


Συνοπτικά μια επεξήγηση για τα περιεχόμενα του φακέλου.
	Οι φάκελοι burst και goodness περιέχουν τουs κώδικες και τα headers αντίστοιχα.
	Τα αρχεία interactive ,non interactive ,interactive -non-interactive  είναι τα 3 προφιλ.Εχω προσθέσει και ένα ακόμα απλό προφιλ προκειμένου να δείξω την αποτυχία της SJF.



Το αρχείο Provlhmatiko_gia_burst είναι ένα ενδεικτικό παράδειγμα αποτυχίας της SJF όπου μια διεργασία που τρέχει για κάποιο χρόνο αποκτάει μεγάλο exp_burst καιστην συνέχεια οι διεργασίες που έρχονται μετα από αυτή τελιώνουν  γρηγορότερα με αποτέλεσμα η πρώτη διεργασία να περιμένει να εκτελεστεί στο τέλος (και εαν έχει απομείνει χρόνος) ,κάτι που δεν ισχύει για τον κώδικα της goodness.
Aπό την άλλη η SJF δείχνει να χειρίζεται σωστά τις αλλαγές μεταξύ interactive - non interactive αφού δίνει προτεραιότητα στις interactive  διεργασίες όταν αυτές χρειαστούν τον επεξεργαστή αφήνοντας τις υπολογιστικές σε 2η μοίρα.
